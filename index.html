What is an artifact?
An artifact is a live, working output that Claude builds and renders right inside the conversation — not just text you copy and paste into something else.
What kinds of artifacts can Claude build?
Formatted documents, slide decks, interactive quizzes, drag-and-drop activities, visual models, error analysis activities, Jeopardy boards, sentence frames, graphic organizers — all from a single prompt.
What do you do with an artifact?
Copy the HTML and paste it into Mindsmith's embed tile, drop it into a Brightspot content block, or hand it to a developer to wrap in a SCORM package for LMS score tracking.

Slide 13 of 16 · Integration
Adding an artifact to Mindsmith
Claude generates the activity. Mindsmith's embed tile does the rest.
  • 1
    Ask Claude to build the activity
    Describe what you need — topic, grade level, activity type. Claude generates it as a live artifact in the conversation.
  • 2
    Copy the HTML from the artifact panel
    Click the download or copy icon at the top of the artifact panel in Claude. The activity is a self-contained HTML file.
  • 3
    Add an embed tile to your Mindsmith lesson
    This is the same embed tile used for web components and YouTube videos.
  • 4
    Paste the HTML into the embed tile
    The activity renders live inside the lesson — no external link, no separate tab.

Slide 15 of 16 · Assessment
Formative vs. graded assessments
Out of the box, Claude's artifacts give learners immediate feedback but don't report scores anywhere. Which approach you need depends on how the activity is being used.
✓  Formative — no score capture needed
Used for practice and self-check. Learners see their own feedback in the moment. No score is sent to a gradebook or LMS. Works as-is in Mindsmith or Brightspot — nothing extra required.
📊  Graded — score capture required
To report scores to an LMS gradebook, the activity needs to be delivered as a SCORM package — a standard format that LMS platforms like Canvas or TalentLMS already understand. See the next slide for how Claude helps with this.
For most practice and engagement use cases, paste Claude's HTML straight into Mindsmith or Brightspot and you're done. SCORM is only needed when scores need to flow into a gradebook.
Slide 16 of 16 · Assessment
Claude and SCORM packages
SCORM is the standard format LMS platforms use to receive and record scores. Claude can generate everything needed to build one.
📄
The activity HTML
The interactive quiz or activity itself — the same artifact you've seen in this presentation.
⚙️
The SCORM connector
JavaScript that captures the learner's score and passes it to the LMS when the activity is submitted.
📋
The manifest file
An XML file that tells the LMS what's in the package and how to launch it.
The one step Claude can't do: A SCORM package is uploaded to an LMS as a .zip file. Claude generates all the files and tells you exactly how to organize and zip them — but the zipping and uploading is a manual step. It takes about two minutes once the files are ready. Testing also requires uploading to an actual LMS; SCORM Cloud offers a free tier specifically for this.
Bottom line: Claude does the heavy lifting on the content and the code. You zip the files and upload. No developer required for a straightforward single-activity package.
